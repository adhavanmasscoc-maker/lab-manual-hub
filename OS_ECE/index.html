EX NO: 1 - INSTALLING WINDOWS 10/11 ON A VIRTUAL MACHINE

AIM

To install and configure Windows 10 or Windows 11 operating system on a Virtual Machine using Oracle VirtualBox, including setting up virtual hardware, attaching installation media, and creating a local user account.

APPARATUS REQUIRED

S.NOCOMPONENTSQUANTITY
1Host Computer1
2Software ToolsWindows 10/11 ISO file, Oracle VirtualBox
3PeripheralsKeyboard & Mouse

PROCEDURE

Step 1: Download Windows ISO File
  1. Visit Microsoft Software Download or Windows 11 page.
  2. Select edition -> Click Confirm.
  3. Choose language -> Click Confirm.
  4. Download 64-bit ISO file (approx. 5-6 GB).
Step 2: Install and Launch VirtualBox
  1. Download and install VirtualBox.
  2. Open VirtualBox.
Step 3: Create a New Virtual Machine
  1. Click "New" on the toolbar.
  2. Enter details:
    • Name: Windows10_VM
    • Machine Folder: Default (or choose your custom folder)
    • Type: Microsoft Windows
    • Version: Windows 10 (64-bit) or Windows 11 (64-bit)
  3. Click Next.
Step 4: Allocate Memory (RAM)
  1. Set RAM size to at least 4096 MB (4GB) or more.
  2. Click Next.
Step 5: Create a Virtual Hard Disk
  1. Choose "Create a virtual hard disk now" -> Click Create.
  2. Choose VDI (VirtualBox Disk Image) -> Click Next.
  3. Choose Dynamically allocated -> Click Next.
  4. Set size to at least 50 GB -> Click Create.
Step 6: Attach Windows ISO to Virtual Machine
  1. Select the created VM -> Click Settings.
  2. Go to Storage tab.
  3. Under Controller: IDE, click the Empty disk icon.
  4. On the right side, click the disk icon -> Choose a disk file.
  5. Browse and select the downloaded Windows ISO file.
  6. Click OK to save settings.
Step 7: Start the VM
  1. Select Windows10_VM -> Click Start.
  2. VM will boot from ISO and load the Windows installer.
Step 8: Begin Installation
  1. Select:
    • Language: English (United States)
    • Time and Currency Format: English (India)
    • Keyboard: US
  2. Click Next.
  3. Click Install Now.
Step 9: Enter Product Key or Skip
  1. If you have a product key, enter it.
  2. Otherwise, click "I don't have a product key" to proceed with limited functionality.
  3. Choose the edition (e.g., Windows 10 Pro) -> Click Next.
Step 10: Accept License and Install
  1. Accept the license agreement -> Click Next.
  2. Choose Custom: Install Windows only (advanced).
  3. Select the virtual hard disk -> Click Next.
Step 11: Complete Setup and User Configuration
  1. Let Windows install and reboot.
  2. On first boot:
    • Choose Region -> Click Yes
    • Choose Keyboard Layout -> Click Yes
    • Skip second keyboard layout (optional)
  3. Create a local account:
    • Username: e.g., StudentUser
    • Password: Choose a secure password (optional)
    • Set Security Questions
  4. Choose Privacy Settings -> Turn off unnecessary options -> Click Next.

RESULT

Windows 10/11 operating system was successfully installed and configured on a Virtual Machine using Oracle VirtualBox. A local user account was created, and the virtual machine is fully operational.

EX NO: 2 - CREATING USER ACCOUNTS AND CONFIGURING PERMISSIONS

AIM

To demonstrate the process of creating new user accounts and assigning appropriate permissions to ensure controlled access to system resources.

APPARATUS REQUIRED

S.NOCOMPONENTSQUANTITY
1Host Computer1
2Software ToolsWindows 10/11
3PeripheralsKeyboard & Mouse

PROCEDURE

Step 1: Open User Management
  1. Press Win + I to open Settings.
  2. Navigate to Accounts -> Select Family & Other Users from the left pane.
Step 2: Add Three Users A. Add Admin User
  1. Click Add someone else to this PC.
  2. In the Microsoft account prompt, click I don't have this person's sign-in information.
  3. Click Add a user without a Microsoft account.
  4. Enter:
    • Username: AdminUser
    • Password: (Create a secure password)
  5. Click Next.
  6. After account is created, click on AdminUser under "Other users" -> Click Change account type.
  7. Set type to Administrator -> Click OK.
B. Add Standard User
  1. Repeat the above steps to create a new user:
    • Username: StandardUser
    • Password: (Enter password)
  2. Do not change the account type — Standard is default.
C. Add Guest User
  1. Again, add a user without Microsoft account.
    • Username: GuestUser
    • Password: (Optional - leave blank or set a simple one)
  2. Leave as Standard type. (Note: Classic Guest account is deprecated; emulate with limited permissions.)
Step 3: Set or Change Passwords (Optional)
  1. Open Control Panel -> Navigate to User Accounts -> Manage another account.
  2. Select each user -> Click Change the password (or create one if not set).
Step 4: Configure Parental Controls for Standard User A. Add Standard User to Family Safety
  1. Open Settings -> Accounts -> Family & Other Users.
  2. Under Your family, click Add a family member.
  3. Add Standard User (or create again under child account type).
  4. Once added, visit https://account.microsoft.com/family and sign in.
  5. Select the Standard User child account -> Configure:
    • Screen time limits
    • App and game restrictions
    • Web filtering
    • Activity reports

RESULT

New user accounts were successfully created in the system. Appropriate permissions and access rights were assigned to each user based on their roles. Users could access only the resources allowed by their permissions.

EX NO: 3 - CREATING NTFS PARTITION AND APPLYING FOLDER-LEVEL PERMISSIONS

AIM

To create an NTFS partition on a storage device and apply folder-level permissions to control and manage user access to files and directories.

APPARATUS REQUIRED

S.NOCOMPONENTSQUANTITY
1Host Computer1
2Software ToolsWindows 10/11
3PeripheralsKeyboard & Mouse

PROCEDURE

Part A: Creating and Formatting NTFS Partition Step 1: Open Disk Management
  1. Press Win + R, type diskmgmt.msc, and press Enter.
  2. The Disk Management console opens, showing all current volumes.
Step 2: Shrink an Existing Volume
  1. Right-click on a partition with free space (usually C:).
  2. Select Shrink Volume.
  3. Enter the amount to shrink (e.g., 10240 MB = 10GB) -> Click Shrink.
  4. A new Unallocated space will appear.
Step 3: Create a New Partition
  1. Right-click the Unallocated space -> Click New Simple Volume.
  2. Click Next in the wizard.
  3. Specify volume size -> Click Next.
  4. Assign a Drive Letter (e.g., E:) -> Click Next.
  5. Choose:
    • File System: NTFS
    • Volume Label: e.g., DATA_PART
    • Enable Perform a quick format
  6. Click Next -> Finish.
Part B: FAT32 vs. NTFS - Comparison Table
Feature FAT32 NTFS
Max File Size 4 GB max > 4 GB
Max Partition Size Up to 32 GB (Windows) Up to 256 TB
Security (Permissions) No Yes
Journaling No Yes
Compression Support No Yes
OS Compatibility Windows, Linux, macOS, etc. Windows, Linux (limited)
Part C: Assign NTFS Folder-Level Permissions Step 1: Create a Folder
  1. Navigate to the newly created partition (e.g., E:).
  2. Right-click -> New -> Folder -> Name it SharedDocs.
Step 2: Open Folder Security Settings
  1. Right-click the SharedDocs folder -> Select Properties.
  2. Go to the Security tab -> Click Edit.
Step 3: Add Users and Set Permissions
  1. Click Add -> Type User1 -> Click Check Names -> OK.
  2. With User1 selected -> Check Read permission -> Apply.
  3. Click Add -> Type User2 -> Click Check Names -> OK.
  4. With User2 selected -> Check Full Control -> Apply -> OK.
  5. (Optional step to add a user: Control Panel -> User Accounts -> Manage another account -> Add new user).

RESULT

An NTFS partition was successfully created and formatted on the storage device. Folder-level permissions were applied to restrict or grant access to specific users or groups.

EX NO: 4 - MANAGING SERVICES WITH DEPENDENCIES

AIM

To study and implement service management with dependencies using a service manager. This includes creating, modifying, and controlling system services to ensure they start, stop, and restart in the correct order based on their dependencies.

APPARATUS REQUIRED

S.NOCOMPONENTSQUANTITY
1Host Computer1
2Software ToolsWindows 10/11
3PeripheralsKeyboard & Mouse

PROCEDURE

Step 1: Open the Services Management Console
  1. Press Win + R on your keyboard to open the Run dialog box.
  2. Type services.msc and press Enter.
  3. The Services window will appear, listing all the background services running on the system.
Step 2: Locate the "Windows Update" Service
  1. In the Services window, scroll down alphabetically to find the service named Windows Update.
  2. Right-click on Windows Update and select Properties.
Step 3: Examine Dependencies
  1. In the Properties window, click on the Dependencies tab.
  2. Observe two sections:
    • This service depends on the following system components (e.g., Remote Procedure Call (RPC))
    • The following system components depend on this service (e.g., Delivery Optimization, BITS, or Update Orchestrator Service)
Step 4: Stop the Service
  1. Go back to the General tab in the same Properties window.
  2. Under Service status, click Stop to halt the service.
  3. Wait a few seconds until the service status changes to "Stopped".
Step 5: Disable the Service
  1. In the same General tab, locate the Startup type drop-down menu.
  2. Change it from Manual or Automatic to Disabled.
  3. Click Apply and then OK to confirm the changes.
Step 6: Observe the System Behavior

After disabling the Windows Update service, expected effects are:

  • Windows will no longer download or install system updates.
  • Microsoft Store apps and drivers may fail to update.
  • Other services relying on update mechanisms (like BITS or Windows Installer) may behave unexpectedly.
  • Security vulnerabilities may remain unpatched.
  • Caution: This setting is not recommended on production systems.
Step 7: Restore the Service (Post-Experiment Cleanup)
  1. Repeat Steps 1–2 to open Windows Update -> Properties.
  2. Change Startup type back to Manual or Automatic (Delayed Start).
  3. Click Start under Service status.
  4. Click Apply -> Click OK.

RESULT

The experiment was successfully completed. Services with dependencies were identified and managed correctly starting in the required order and stopping gracefully when required.

EX NO: 5 - EXPLORING LINUX DIRECTORIES AND COMMANDS

AIM

To understand the structure of the Linux file system by exploring directories using basic Linux commands and to learn how to navigate, list, and manipulate files and directories from the command line.

APPARATUS REQUIRED

S.NOCOMPONENTSQUANTITY
1Host Computer1
2Software ToolsLinux OS
3PeripheralsKeyboard & Mouse

PROCEDURE

Step 1: Open the Terminal
  1. Press Ctrl + Alt + T (or use the terminal app in GUI).
  2. Make sure the shell prompt ($) is visible.
Step 2: View Key System Directories

Run the following command to list the contents of core system directories:

ls /bin /sbin /etc /dev /home /tmp /var /usr
  • /bin - Basic user command binaries (e.g., ls, cp, rm)
  • /sbin - System binaries (e.g., system administration commands like shutdown)
  • /etc - Configuration files for all programs and services
  • /dev - Device files (e.g., /dev/sda, /dev/null)
  • /home - User home directories (e.g., /home/student)
  • /tmp - Temporary files
  • /var - Variable files (e.g., logs, cache, mail)
  • /usr - Secondary hierarchy for read-only user data (e.g., programs, libraries)
Step 3: Visualize Directory Tree Structure (Optional)

Use the tree command to visualize a folder structure:

sudo apt install tree   # Only if 'tree' is not already installed
tree /etc | head -n 30  # Limit output for readability

This displays a hierarchical tree of files and subdirectories under /etc, showing configuration file locations.

Step 4: Check Disk Space Usage

Run the following command to see disk usage by mounted file systems:

df -h
  • Shows space used and available on mounted partitions in human-readable format.
  • Columns: Filesystem, Size, Used, Avail, Use%, Mounted on
Step 5: Check Directory Sizes Inside /home

This command lists disk space usage for each user's home directory:

du -sh /home/*
  • -s : summarize only
  • -h : human-readable (e.g., MB, GB)
  • Output: Each line shows space used by each user's home directory.
Summary of Directory Purposes
DirectoryPurpose
/etcContains system-wide configuration files
/binEssential command binaries for users
/sbinSystem administration binaries
/devDevice files representing hardware
/homeUser home directories
/tmpTemporary files, cleared on reboot
/varVariable files (logs, cache, spool)
/usrRead-only user programs, libraries, docs

RESULT

Successfully navigated through various Linux directories using commands like cd, ls, and pwd. Gained knowledge of the Linux directory hierarchy and common directory contents.

pwd

Verify:

ls lab1/lab2

Remove the file from lab2:

rm lab1/lab2/file1.txt

Check that it's deleted:

EX NO: 7 - INSTALLING VLSI TOOLS ON LINUX

AIM

To successfully install and configure VLSI (Very Large-Scale Integration) design tools on a Linux operating system, enabling the environment to perform integrated circuit design, simulation, and verification tasks.

APPARATUS REQUIRED

S.NOCOMPONENTSQUANTITY
1Host Computer1
2Software ToolsLinux OS
3PeripheralsKeyboard & Mouse

PROCEDURE

Step 1: Update the System

Update the system package list to ensure all repositories are up to date:

sudo apt update
Step 2: Install Compilation Tools

Install essential tools like gcc, g++, and make for compiling VHDL and C/C++ programs:

sudo apt install gcc g++ make -y
Step 3: Install GHDL and GTKWave

Install open-source VHDL simulator GHDL and waveform viewer GTKWave:

sudo apt install ghdl gtkwave -y

These tools are typically available in the default Ubuntu repositories.

Step 4: Configure Environment Variables (if needed)

This step is only required if tools are installed to custom locations. If using default apt-based install, skip to Step 5.

  1. Open the .bashrc file in the home directory: 
    nano ~/.bashrc
  2. Add the following line at the end of the file (adjust path as needed): 
    export PATH=$PATH:/usr/local/bin/ghdl:/usr/local/bin/gtkwave
  3. Save and exit:
    • Press Ctrl + O to save
    • Press Enter
    • Press Ctrl + X to exit
  4. Reload the configuration: 
    source ~/.bashrc
Step 5: Verify Installation

Check the installed versions to confirm tools are working:

ghdl --version
gtkwave --version

Sample Output:

GHDL 3.0.0 (Ubuntu 22.04)
GTKWave Analyzer v3.3.105

RESULT

The VLSI tools were successfully installed on the Linux system, and their basic functionalities were verified. The environment was set up correctly.

EX NO: 8 - SHELL SCRIPT USING IF STATEMENT

AIM

To write and execute a shell script that uses IF statements to make decisions based on conditions, demonstrating the use of conditional logic in shell scripting.

APPARATUS REQUIRED

S.NOCOMPONENTSQUANTITY
1Host Computer1
2Software ToolsLinux OS
3PeripheralsKeyboard & Mouse

PROCEDURE

Step 1: Open the Terminal

Launch your terminal using Ctrl + Alt + T or from the applications menu.

Step 2: Create a New Script File

Create a shell script file using nano or any text editor:

nano test_if.sh
Step 3: Write the Script

Paste the following script content:

#!/bin/bash
echo "Enter number:"
read n
if [ $n -gt 10 ]
then
    echo "Greater than 10"
else
    echo "10 or less"
fi

Save the script: Press Ctrl + O, then Enter to save. Press Ctrl + X to exit the editor.

Step 4: Make the Script Executable

Run the following command to add execution permission:

chmod +x test_if.sh
Step 5: Execute the Script

Run the script using:

./test_if.sh

Example Run:

$ ./test_if.sh
Enter number:
15
Greater than 10

RESULT

The shell script using if statements was successfully created and executed. It correctly evaluated the given conditions and performed actions based on those conditions.

EX NO: 9 - SHELL SCRIPT USING WHILE STATEMENT

AIM

To develop and execute a shell script using the while loop in Bash that demonstrates the concept of iterative execution by printing a sequence of numbers from 1 to 5.

APPARATUS REQUIRED

S.NOCOMPONENTSQUANTITY
1Host Computer1
2Software ToolsLinux OS
3PeripheralsKeyboard & Mouse

PROCEDURE

Step 1: Open Terminal
  • Start your Linux system.
  • Open the terminal using Ctrl + Alt + T or from the applications menu.
Step 2: Create a New Shell Script File
  • Use a text editor like nano, vi, or gedit to create a shell script.
  • Example using nano: 
    nano while_loop.sh
Step 3: Write the Shell Script Code

Enter the following sample script in the editor:

#!/bin/bash

# Initialize the counter variable i=1
i=1

# Start while loop
while [ $i -le 5 ]
do
    echo "Number: $i"
    i=$((i + 1))
done

This script prints numbers from 1 to 5 using a while loop.

Step 4: Save and Exit the Editor
  • In nano: Press Ctrl + O -> Enter to save, and Ctrl + X to exit.
Step 5: Give Execute Permissions to the Script

Make the script executable using the chmod command:

chmod +x while_loop.sh
Step 6: Run the Script

Execute the script in the terminal:

./while_loop.sh

You should see:

Number: 1
Number: 2
Number: 3
Number: 4
Number: 5
Step 7: Modify the Script (Optional)

Try modifying the script to read user input in a loop, perform a countdown, or run until the user enters a specific value (e.g., exit).

Example:

#!/bin/bash
input=""
while [ "$input" != "exit" ]
do
    echo "Enter something (type 'exit' to quit):"
    read input
    echo "You entered: $input"
done

RESULT

The shell script using while statements was successfully created and executed. It correctly evaluated the given conditions and performed actions based on those conditions.
