EX NO: 1 - AI Program To Design A Beam

AIM

Use a simple AI program to design a beam that uses the least amount of material but is still strong enough.

ALGORITHM

  1. Start the program.
  2. Define the beam parameters such as length of the beam, applied load, and allowable maximum stress.
  3. Initialize a fitness function to calculate the material volume of the beam based on width, height, and length.
  4. Compute the bending stress using the moment of inertia of the rectangular beam section.
  5. Apply a penalty to the fitness value if the calculated stress exceeds the safe stress limit.
  6. Initialize the beam dimensions with an initial width and height.
  7. Iteratively modify the beam dimensions using random variations (simple genetic search).
  8. Restrict the dimensions within practical limits to avoid unrealistic values. Evaluate each new beam design using the fitness function.
  9. Update the best beam dimensions if a design with lower material usage and safe stress is found.
  10. Repeat the optimization process for a fixed number of generations.
  11. Display the optimal beam dimensions and minimum material required.

PROGRAM

import random
import numpy as np

# Simple beam (100cm long, 10kN load at end)
L = 100 # length cm
P = 10000 # load N
sigma_max = 1000 # safe stress N/cm²

def fitness(w, h):
    # Material = width * height
    material = w * h * L
    # Stress check
    I = (w * h**3) / 12.0 # moment of inertia
    stress = (P * L * (h / 2)) / I
    
    # Penalty if too weak
    if stress > sigma_max:
        material *= 10 # big penalty
    return material

# Simple GA
best_w, best_h = 10, 10
best_material = fitness(10, 10)

for generation in range(1000):
    # Try random changes
    w = best_w + random.uniform(-2, 2)
    h = best_h + random.uniform(-1, 3)
    
    w = max(2, min(20, w)) # keep reasonable
    h = max(3, min(25, h))
    
    material = fitness(w, h)
    
    if material < best_material:
        best_w, best_h = w, h
        best_material = material

# Results
print(f"Best beam: {best_w:.1f} cm wide x {best_h:.1f} cm tall")
print(f"Material needed: {best_material:.0f} cm³")
print("Ready to build!")

OUTPUT

Best beam: 2.0 cm wide x 3.0 cm tall
Material needed: 6000 cm³
Ready to build!

RESULT

The AI-based optimization successfully identified an efficient beam design that minimizes material usage while maintaining structural safety. Thus, the beam design obtained through the AI approach satisfies strength constraints with minimum material consumption, demonstrating effective optimization.

To write and execute a neural network to guess how much load a pile (foundation) can carry based on soil type, pile size, etc.

  1. Start the program.
  2. Import required libraries such as NumPy, Pandas, and MLPRegressor from Scikit-learn.
  3. Create a dataset containing pile diameter, pile length, soil type, embedment ratio, and corresponding load-carrying capacity.
  4. Separate the dataset into input features (diameter, length, soil type, embedment ratio) and output target (load capacity).
  5. Initialize a neural network regression model with two hidden layers.
  6. Train the neural network using the given pile foundation dataset.
  7. Evaluate the trained model using the R² score.

PROGRAM

import numpy as np
from sklearn.neural_network import MLPRegressor
import pandas as pd

# pile data (typical soils)
data = pd.DataFrame({
    'diameter': [30, 45, 60, 30, 45, 60, 45, 60, 30, 45], # cm
    'length': [500, 600, 800, 400, 500, 700, 600, 900, 500, 600], # cm
    'soil_type': [1,1,1,2,2,2,3,3,3,1], # 1=clay, 2=sand, 3=gravel
    'embed_ratio': [10,12,13,8,10,12,12,15,10,11], # L/diameter

# Simple Neural Network

# Predict new pile

prediction = nn.predict(new_pile)[0]
print(f"\n50cm dia 7m long pile in SAND: {prediction:.0f} kN capacity")

# Test different soils
for soil in [1, 2, 3]:
    test = np.array([[50, 700, soil, 14]])
    cap = nn.predict(test)[0]
    soil_name = ['Clay', 'Sand', 'Gravel'][soil-1]
    print(f" {soil_name}: {cap:.0f} kN")

EX NO: 3 - Fuzzy Logic

AIM

Create a fuzzy logic system that tells whether a road is in "Good", "Okay", or "Bad" condition based on things like cracks and bumps.

ALGORITHM

  1. Start the program.
  2. Define fuzzy membership functions for cracks to classify them as low or high severity.
  3. Define fuzzy membership functions for bumps to classify them as low, medium, or high severity.
  4. Accept the input values for crack severity and bump severity.
  5. Calculate the membership values for cracks and bumps using the fuzzy functions.
  6. Apply fuzzy IF-THEN rules to evaluate road condition as Good, Okay, or Bad.
  7. Compute the strength of each rule using minimum and maximum operators.
  8. Select the road condition with the highest rule strength.
  9. Display the final road condition result. Stop the program.

PROGRAM

# Simple Fuzzy Logic Road Condition System
# Inputs: cracks (0-10), bumps (0-10) -> Output: "Good", "Okay", "Bad"

def fuzzy_cracks(severity):
    low = max(0, min(1.0, (10 - severity) / 10.0)) # 0 cracks = 1.0
    high = max(0, severity / 10.0) # 10 cracks = 1.0
    return low, high

def fuzzy_bumps(severity):
    low = max(0, min(1.0, (8 - severity) / 8.0)) # 0 bumps = 1.0
    med = max(0, min(1.0, severity / 5.0)) * min(1.0, (10 - severity) / 5.0)
    high = max(0, (severity - 6) / 4.0) # 10 bumps = 1.0
    return low, med, high

def fuzzy_decision(c_low, c_high, b_low, b_med, b_high):
    # Rules (simple IF-THEN)
    good = min(c_low, b_low) # Few cracks + few bumps
    okay = max(min(c_low, b_med), min(c_high, b_low)) # Either okay
    bad = min(c_high, b_high) # Many cracks + many bumps
    
    # Pick strongest rule
    scores = {'Good': good, 'Okay': okay, 'Bad': bad}
    return max(scores, key=scores.get)

# Test it!
cracks = 3 # few cracks
bumps = 7 # many bumps
c_low, c_high = fuzzy_cracks(cracks)
b_low, b_med, b_high = fuzzy_bumps(bumps)

result = fuzzy_decision(c_low, c_high, b_low, b_med, b_high)
print(f"Cracks={cracks}, Bumps={bumps} -> Road: {result}")

OUTPUT

Cracks=3, Bumps=7 -> Road: Okay

RESULT

The fuzzy logic system successfully evaluated the road condition based on crack and bump severity. Thus, the fuzzy logic approach effectively assesses road condition using imprecise inputs.

# SUPER SIMPLE Reservoir Controller
# NN predicts demand + Fuzzy releases water
import random

# 1. NN (fake - predicts demand from rain/temp)
def predict_demand(rain, temp):
    demand = 500 + rain * 10 + temp * 20 # Simple formula
    return min(demand, 2000)

# 2. Fuzzy decides % to release (0-100%)
def fuzzy_release(fullness, inflow, demand):
    # Fullness/Inflow/Demand -> LOW(0), MED(1), HIGH(2)
    f = 0 if fullness < 0.4 else 1 if fullness < 0.8 else 2
    i = 0 if inflow < 1000 else 1 if inflow < 4000 else 2
    d = 0 if demand < 800 else 1 if demand < 1500 else 2
    
    # Simple rules
    if f == 2 and i == 2: return 80 # Full + lots inflow = release!
    if f == 0: return 10 # Empty = conserve
    if d == 2: return 60 # High demand = release more
    return 30 # Default

# Test
storage = 80000 # 80% full (max 100k)
inflow = 3500 # m³/day
rain, temp = 25, 33

demand = predict_demand(rain, temp)
release_pct = fuzzy_release(storage / 100000, inflow, demand)
release = (release_pct / 100.0) * storage / 1000.0 # 1000 liters/m³

print(f"Rain {rain}mm, Temp {temp}°C")
print(f"Demand: {demand} m³")
print(f"Release {release_pct}% = {release:.0f} thousand liters")

Use a genetic algorithm to design a water pipe system that works well and costs less.

ALGORITHM

  3. Specify the required water demand and pipe length.
  4. Define a fitness function to calculate the total cost of a selected pipe.

  6. Initialize the best pipe choice and calculate its fitness value.
  7. Randomly select pipe options for a fixed number of iterations (genetic search).
  8. Evaluate the fitness of each selected pipe option.
  9. Update the best pipe choice if a lower-cost feasible solution is found.
  10. Display the optimal pipe diameter, total cost, and flow capacity. Stop the program.

PROGRAM

# Simple GA Water Pipe Optimizer
# Finds cheapest pipes that still deliver water
import random

# Pipe options: [diameter_cm, cost_per_m, flow_capacity_m3h]
pipes = [
    [10, 20, 5],   # small pipe
    [15, 35, 12],  # medium
    [20, 55, 22],  # large
    [25, 80, 35]   # extra large
]

demand = 15 # m³/hour needed
length = 100 # meters

def fitness(pipe_choice):
    # total cost
    cost = pipes[pipe_choice][1] * length
    # penalty if can't deliver demand
    flow = pipes[pipe_choice][2]
    if flow < demand:
        cost += 10000 # huge penalty
    return cost

# GA magic
best_pipe = 0
best_cost = fitness(0)

for _ in range(1000): # 1000 tries
    test_pipe = random.randint(0, 3)
    test_cost = fitness(test_pipe)
    
    if test_cost < best_cost:
        best_pipe = test_pipe
        best_cost = test_cost

d, c, f = pipes[best_pipe]
print(f"Use {d}cm pipes")
print(f"Cost: {c * length}")
print(f"Flow: {f} m³/h ✓")
print("Cheapest that works!")

OUTPUT

Use 20cm pipes
Cost: 5500
Flow: 22 m³/h ✓
Cheapest that works!

RESULT

The genetic algorithm successfully identified the most cost-effective pipe that satisfies the required water demand, demonstrating the application of genetic algorithms in water distribution system optimization.

EX NO: 6 - Genetic Algorithm To Make A Smart Schedule For A Construction Project

AIM

Use a genetic algorithm to make a smart schedule for a construction project so it gets done faster.

ALGORITHM

  1. Start the program.
  2. Define the list of construction tasks along with their duration and required number of workers.
  3. Specify the total number of workers available for the project.
  4. Define a fitness function to calculate the total project duration.
  5. Add a penalty in the fitness function if the number of workers required exceeds the available workers.
  6. Initialize an initial task execution order and calculate its fitness value.
  7. Generate new task orders by randomly swapping two tasks (genetic operation).
  8. Evaluate the fitness of each new task order.
  9. Update the best task order if a schedule with fewer total days is obtained.
  10. Repeat the process for a fixed number of iterations.
  11. Display the optimized task schedule and total project duration. Stop the program.

PROGRAM

# Simple GA Construction Scheduler
# Makes project FASTER by smart task ordering
import random

# Tasks: [Name, duration_days, needs_worker]
tasks = [
    ["Foundation", 5, 2],
    ["Columns", 4, 3],
    ["Slab", 6, 4],
    ["Walls", 3, 2],
    ["Roof", 2, 1],
    ["Finish", 4, 2]
]

workers = 4 # total available

def fitness(order):
    total_days = 0
    used_workers = 0
    for task_id in order:
        task_days, task_workers = tasks[task_id][1], tasks[task_id][2]
        total_days += task_days
        used_workers = max(0, used_workers + task_workers - workers)
    return total_days + (used_workers * 10) # penalty for overwork

# GA finds best order
best_order = list(range(len(tasks)))
best_days = fitness(best_order)

for _ in range(2000):
    # Swap random tasks
    test_order = best_order[:]
    i, j = random.sample(range(len(test_order)), 2)
    test_order[i], test_order[j] = test_order[j], test_order[i]
    
    test_days = fitness(test_order)
    if test_days < best_days:
        best_order = test_order
        best_days = test_days

# Results
print("SMART SCHEDULE:")
for i, task_id in enumerate(best_order):
    print(f"Day {i+1}: {tasks[task_id][0]}")
print(f"\nTotal: {best_days} days (faster!)")

OUTPUT

Day 1: Foundation
Day 2: Columns
Day 3: Slab
Day 4: Walls
Day 5: Roof
Day 6: Finish

Total: 24 days (faster!)

RESULT

The genetic algorithm successfully optimized the construction task sequence to reduce the overall project duration. Thus, the genetic algorithm effectively improved construction scheduling by minimizing project time while considering workforce constraints.

PROGRAM

// IoT Soil Moisture Auto-Watering System
// Checks soil -> turns pump ON/OFF + phone alerts
#include <ESP8266WiFi.h>
#include <BlynkSimpleEsp8266.h>

char auth[] = "YOUR_BLYNK_TOKEN"; // Blynk app
char ssid[] = "YOUR_WIFI";
char pass[] = "YOUR_WIFI_PASS";

int sensorPin = A0; // Soil moisture sensor
int pumpPin = D1;   // Pump relay
int dryLevel = 600; // Dry threshold

void setup() {
    Serial.begin(9600);
    Blynk.begin(auth, ssid, pass);
    pinMode(pumpPin, OUTPUT);
    digitalWrite(pumpPin, LOW);
}

void loop() {
    Blynk.run();
    int moisture = analogRead(sensorPin);
    
    // Send to phone
    Blynk.virtualWrite(V0, moisture);
    
    if (moisture > dryLevel) {
        digitalWrite(pumpPin, HIGH); // DRY -> WATER!
        Blynk.virtualWrite(V1, 1);
        Serial.println("WATERING");
    } else {
        digitalWrite(pumpPin, LOW); // WET -> STOP
        Blynk.virtualWrite(V1, 0);
        Serial.println("Soil good");
    }
    delay(3000);
}

EX NO: 8 - Detect shake in a building using sensors and IoT

AIM

Use sensors and IoT to watch for shakes or movements in a building to detect any damage early.

ALGORITHM

  1. Start the program.
  2. Include required libraries for I2C communication and the MPU6050 accelerometer.
  3. Define the shake threshold value for detecting abnormal building vibrations.
  4. Initialize serial communication and the MPU6050 sensor.
  5. Continuously read acceleration values along the X, Y, and Z axes.
  6. Calculate the resultant acceleration magnitude from the three-axis data.
  7. Compare the calculated acceleration with the predefined shake threshold.
  8. If the acceleration exceeds the threshold, generate an alert message.
  9. If the acceleration is within the limit, indicate that the building is stable.
  10. Display the vibration status on the serial monitor. Repeat process. Stop the program.

PROGRAM

// SIMPLE Building Shake Detector
#include <Wire.h>
#include <MPU6050.h>

MPU6050 mpu;
float shakeLimit = 1.5; // Alert threshold

void setup() {
    Serial.begin(9600);
    Wire.begin();
    mpu.initialize();
    Serial.println("Building monitor ON");
}

void loop() {
    int16_t ax, ay, az;
    mpu.getAcceleration(&ax, &ay, &az);
    
    // Calculate total magnitude
    float accel = sqrt((float)ax*ax + (float)ay*ay + (float)az*az) / 16384.0;
    
    Serial.print("Shake: ");
    Serial.println(accel);
    
    if (accel > shakeLimit) {
        Serial.println("ALERT! Building shaking!");
    } else {
        Serial.println("Stable");
    }
    delay(1000);
}
