📘 UNIT 1: Introduction to AI and Problem Solving

🧠 Memory Trick (Agent Types): S-M-G-U-L (Simple, Model, Goal, Utility, Learning)

📗 2 Mark Questions

Q: Define Artificial Intelligence.

Ans:
Definition: AI is the ability of a machine to perform tasks that require human intelligence, such as learning, reasoning, and problem-solving.
🟡 Key Point: AI aims to build rational agents that act to achieve the best outcome.

Q: What is an AI Agent?

Ans:
Definition: An entity that perceives its environment through sensors and acts upon it through actuators.
🟡 Key Point: Agent = Architecture + Program.

Q: Differentiate between BFS and DFS.

Ans:
Definition: BFS expands level-by-level using a Queue (FIFO). DFS expands branch-by-branch using a Stack (LIFO).
🟡 Key Point: BFS guarantees the shortest path, while DFS saves memory.

Q: Define Heuristic Function (h(n)).

Ans:
Definition: An estimate of the cheapest path cost from the current node to the goal node.
🟡 Key Point: Used in informed searches like A* to guide the algorithm efficiently.

📙 16 Mark Answers

🔥 Q1: Explain Agent Structure and its Types in detail. (16 Marks)

  1. Definition: An agent program maps percepts (inputs) to actions (outputs) and runs on a computing architecture.
  2. Explanation (Step-by-step):
    • Simple Reflex Agents: Act only on current percepts. Uses IF-THEN condition-action rules.
    • Model-Based Reflex Agents: Maintain internal state to track parts of the environment they cannot currently see.
    • Goal-Based Agents: Combine state information with goal information to choose actions that achieve the goal.
    • Utility-Based Agents: Choose actions that maximize a "happiness" or efficiency metric.
    • Learning Agents: Can improve their performance over time through a critic and learning element.
  3. Diagram: Draw a box labeled "Agent" with "Sensors" taking in "Percepts" from the "Environment", and "Actuators" sending "Actions" back.
  4. Example: A self-driving car acting as a Model-Based agent navigating traffic.
  5. ⭐ Advantages: Modular, scalable, and capable of handling dynamic environments.
  6. ⚠ Disadvantages: Complex agents require high computational power to process utility functions in real-time.
  7. 🌍 Applications: Autonomous vehicles, NPC behavior in video games, Smart home systems.
  8. ✅ Conclusion: Choosing the right agent architecture depends entirely on whether the environment is observable, deterministic, and static.

🔥 Q2: Explain Constraint Satisfaction Problem (CSP) and Backtracking. (16 Marks)

  1. Definition: A problem defined by a set of variables, a domain of values for each variable, and constraints that limit allowable value combinations.
  2. Explanation (Step-by-step):
    • Define Variables (X).
    • Define Domains (D) - possible values.
    • Define Constraints (C) - rules (e.g., X1 != X2).
    • Backtracking Search: Assign a value to a variable. Check constraints. If valid, move to the next variable. If invalid, go back (backtrack) and try a different value.
  3. Diagram: Draw a search tree where branches represent assignments. Show a red cross "X" where a constraint fails, indicating a backtrack.
  4. Example: The 8-Queens Problem (placing 8 queens on a chessboard so none attack each other).
  5. ⭐ Advantages: Prunes huge portions of the search space early, saving computation time.
  6. ⚠ Disadvantages: Can still be slow (exponential time) for extremely large, heavily constrained problems without good heuristics.
  7. 🌍 Applications: Timetable scheduling, Map coloring, Sudoku solvers.
  8. ✅ Conclusion: CSPs shift the focus from finding a path to simply finding a state that satisfies all logical rules.

📘 UNIT 2: Knowledge Representation

🧠 Memory Trick (KR Types): S-F-C-S (Semantic Nets, Frames, Conceptual Dependency, Scripts)

📗 2 Mark Questions

Q: What is Knowledge Representation (KR)?

Ans:
Definition: The method of structuring real-world information so that an AI system can reason and solve problems.
🟡 Key Point: Transforms data into actionable intelligence.

Q: What are Semantic Networks?

Ans:
Definition: A graphical representation of knowledge using nodes (objects) and edges (relationships like IS-A, HAS-A).
🟡 Key Point: Easy for humans to read but lacks formal logical semantics.

📙 16 Mark Answers

🔥 Q1: Explain Knowledge Representation using Frames. (16 Marks)

  1. Definition: A data structure used to divide knowledge into substructures representing stereotyped situations.
  2. Explanation (Step-by-step):
    • Similar to Object-Oriented Programming (OOP) classes.
    • Consists of Slots (attributes like "Color" or "Legs").
    • Contains Fillers (values like "Red" or "4").
    • Supports Inheritance (a "Car" frame inherits properties from a "Vehicle" frame).
    • Includes Facets (default values, procedures like IF-NEEDED).
  3. Diagram: Draw a table with Frame Name at the top, followed by rows of Slots and their corresponding Fillers.
  4. Example: A "Student" frame with slots for Roll No, Name, and Default University (CIT).
  5. ⭐ Advantages: Groups related knowledge naturally; excellent for handling default expectations.
  6. ⚠ Disadvantages: Rigid structure; difficult to handle exceptions or highly dynamic data.
  7. 🌍 Applications: Database schemas, Expert systems, Natural Language Understanding.
  8. ✅ Conclusion: Frames bridge the gap between human cognitive categorization and machine memory organization.

📘 UNIT 3: Fuzzy and Predictive Logic

🧠 Memory Trick (Chaining): Forward = Data to Goal. Backward = Goal to Data.

📗 2 Mark Questions

Q: What is First-Order Logic (FOL)?

Ans:
Definition: A formal logic system using variables, predicates, and quantifiers (Universal & Existential).
🟡 Key Point: More expressive than Propositional logic.

Q: What is a Fuzzy Set?

Ans:
Definition: A mathematical set without a crisp boundary, where elements have a degree of membership between 0 and 1.
🟡 Key Point: Handles vague human concepts like "tall" or "hot".

📙 16 Mark Answers

🔥 Q1: Explain Forward and Backward Chaining. (16 Marks)

  1. Definition: Two inference algorithms used in rule-based expert systems to deduce conclusions.
  2. Explanation (Step-by-step):
    • Forward Chaining: Data-driven. Starts with known facts. Triggers rules whose premises match the facts. Adds new conclusions to the facts base. Repeats until the goal is found.
    • Backward Chaining: Goal-driven. Starts with the hypothesis (goal). Finds rules that can conclude this goal. Checks if the premises are known facts. If not, sets them as new sub-goals.
  3. Diagram: Draw flowcharts: [Facts -> Rules -> Goal] for Forward, and [Goal -> Rules -> Facts] for Backward.
  4. Example: Forward: Smoke sensor detects smoke -> Triggers Fire Alarm rule. Backward: Doctor suspects cancer (Goal) -> Orders blood tests to verify premises.
  5. ⭐ Advantages: Forward is excellent for real-time monitoring. Backward is highly efficient for targeted queries.
  6. ⚠ Disadvantages: Forward can generate massive amounts of useless facts. Backward fails if the goal is completely unknown.
  7. 🌍 Applications: Production systems, Prolog programming, Medical diagnosis systems.
  8. ✅ Conclusion: The choice depends on the problem: use Forward when data is abundant, and Backward when you have a specific hypothesis to prove.

Ans:
Definition: Open Source Computer Vision Library used for real-time image and video processing.
🟡 Key Point: Used for face detection and image filtering.

📙 16 Mark Answers

🔥 Q1: Explain NLP Preprocessing and Pre-trained Models (BERT). (16 Marks)

  1. Definition: The pipeline of cleaning raw text data and feeding it into advanced deep learning models to extract meaning.
  2. Explanation (Step-by-step):
    • Text Cleaning: Converting to lowercase, removing punctuation and special characters.
    • Tokenization: Splitting sentences into individual words or sub-words.
    • Stopword Removal: Filtering out common words (e.g., "is", "the") that carry little meaning.
    • Pre-trained Models (BERT): Bidirectional Encoder Representations from Transformers. Instead of training from scratch, developers use this pre-trained model and fine-tune it for specific tasks.
  3. Diagram: Draw a pipeline block diagram: Raw Text -> Cleaning -> Tokenization -> BERT Model -> Output (Sentiment/Intent).
  4. Example: Classifying a Google Review as Positive or Negative.
  5. ⭐ Advantages: BERT understands context from both left-to-right and right-to-left simultaneously, providing massive accuracy leaps.
  6. ⚠ Disadvantages: Highly resource-intensive; requires significant GPU power.
  7. 🌍 Applications: Chatbots, Search Engines, Auto-correct, Translation.
  8. ✅ Conclusion: Pre-trained models like BERT have revolutionized NLP by making advanced context understanding accessible to developers.

📗 2 Mark Questions

Q: How is AI used in Healthcare?

Ans:
Definition: AI is used for analyzing medical images (X-rays/MRIs) and predicting disease outbreaks using patient records.
🟡 Key Point: Acts as a highly accurate second opinion for doctors.

Q: What is a Robo-Advisor?

Ans:
Definition: An AI-driven financial platform that provides automated, algorithmic portfolio management.
🟡 Key Point: Minimizes human intervention and lowers investment costs.

📙 16 Mark Answers

🔥 Q1: Discuss Societal Concerns and Ethics in AI. (16 Marks)

  1. Definition: The moral implications, biases, and societal risks associated with deploying AI systems at a global scale.
  2. Explanation (Step-by-step):
    • Bias & Fairness: AI models trained on historical human data often inherit racist or sexist biases (e.g., biased hiring algorithms).
    • Privacy: Massive data scraping, surveillance, and deepfakes violate personal consent.
    • Job Displacement: Automation threatens cognitive and manual jobs across various sectors.
    • Transparency ("Black Box"): Deep learning models are unexplainable. We know the output, but not how the AI arrived at the decision.
  3. Diagram: Draw a structure showing the "Four Pillars of Ethical AI": Fairness, Privacy, Transparency, Accountability.
  4. Example: A facial recognition software failing to accurately identify minorities, leading to false arrests.
  5. ⭐ Advantages of Ethical AI: Builds consumer trust and ensures legal/regulatory compliance.
  6. ⚠ Disadvantages (of ignoring ethics): Massive lawsuits, brand destruction, and severe societal harm.
  7. 🌍 Applications: AI Auditing tools, Explainable AI (XAI) frameworks in healthcare.
  8. ✅ Conclusion: Strict ethical frameworks and government regulations must be integrated during AI development, not as an afterthought.
